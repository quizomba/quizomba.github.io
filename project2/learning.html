What I got from it?

To me, this project was a crazy journey. I am no programmer. My coding knowledge is limited to a month-old Codecademy account. So figuring out how to make things work and change what I wanted to change was quite a challenge. Also, I am not a math-head. I did take some stats classes back in my Bachelor, but it was more than 5 years ago. Cleaning data was also interesting because of data. Anyway, I am proud of what you see here. This is certainly not the most beautiful, or crafty visualization (there are great people in our class). But still: I did it myself, relying on Google, RedBull, and long nights. . And I learnt a lot while doing it.


Here are some thing I learned and others which I could have done better:
  • D3.js
    This was my first take on D3.js. It was not easy, but no doubt now I know way more than a couple of weeks ago. I feel I can really do some things on my own.

  • Time Scale
    There is a bunch of features I wish my visualization had. For example, it would be way better if my visualization had a time scale, with which the user could slide to see changes overtime. I was not able to figure this out, so creating different html with different data was the best option I manage. Not even close to the best possibility.

  • CSS+HTML
    Aside from playing around on FrontPage in Windows95, I had tried designed a website. To make this work, I learnt a considerable amount of CSS and HTML. If this page looks awful is because my configs probably are badly-structured. I am 120% my code is super messy.

  • Data Treatment
    Dealing with data was complicated. I am not quite sure I did things the right way. I should have invested more time here.

  • Exploring the Database
    I should have played more with the data before starting the project. Figure out what variables were most interesting, which had more countries, how did they were collected. This would led me to a more interesting hypothesis.

  • World Mapping is Hard
    One of my ideas was to have a global map, to situate the user better. I tried a lot of different approaches, but could not manage to make this work. This is a weakness in the project.

  • Time Managment
    I postponed this project more than I should. I should have started it earlier. This would have given me more time to explore databases, look for help with implementations, and make things prettier.
